# Components

Компоненты — это основа любого приложения на React. Это небольшие, переиспользуемые кусочки кода, которые управляют отображением и поведением пользовательского интерфейса.
Во многом компоненты ведут себя как обычные функции JavaScript. Они принимают произвольные входные данные (так называемые «пропсы») и возвращают React-элементы, описывающие, что мы хотим увидеть на экране.

### Виды компонентов.
В React есть два вида компонентов:

1. **Функциональные компоненты**

Функциональные компоненты — это простые функции, которые принимают props и возвращают JSX. Они являются наиболее современным и популярным способом написания компонентов.

<CodeBlock code={`function Greeting() {
    return <h1>Привет, Pavel!</h1>;
}

<Greeting />;`} language="javascript" />

2. **Классовые компоненты**

Классовые компоненты — это компоненты, которые создаются с использованием классов. Они использовались до появления хуков, но до сих пор встречаются в старых проектах.


<CodeBlock code={`import React, { Component } from 'react';

  class Greeting extends Component {
    render() {
      return <h1>Привет, Pavel!</h1>;
    }
  }

<Greeting />;`} language="javascript" />

### Основные характеристики компонентов

* **Независимость**

Каждый компонент работает независимо от других. Это значит, что изменения в одном компоненте не затронут другие.

* **Переиспользуемость**

Компоненты можно использовать повторно в разных частях приложения, что снижает количество дублирующегося кода.

* **Иерархия**

Компоненты могут быть вложенными: родительский компонент передаёт данные дочерним через props.

<CodeBlock code={`function ParentComponent() {
    return (
      <div>
        <ChildComponent message={"Привет от родителя!"} />
      </div>
    );
}

function ChildComponent({ message }) {
    return <p>{message}</p>;
}`} language="javascript" />

### Компонента должна себя вести как чистая функция.

**“Чистой”** называется функция, которая:

1. Для одинаковых входных данных всегда возвращает один результат.
2. Не имеет побочных эффектов (то есть не изменяет внешние состояния).
3. Не зависит от внешних состояний.

### JSX и компоненты.

JSX (от JavaScript XML) делает код React читаемым и похожим на HTML.

- Каждый компонент возвращает JSX.
- JSX может содержать другие компоненты, элементы HTML или JavaScript.

<CodeBlock code={`function App() {
    return (
      <div>
        <Header />
        <MainContent />
        <Footer />
      </div>
    );
  }

  function Header() {
    return <h1>Это заголовок</h1>;
  }

  function MainContent() {
    return <p>Основной текст страницы</p>;
  }

  function Footer() {
    return <footer>Здесь футер</footer>;
  }`} language="javascript" />

### Stateless VS Stateful

Stateless компоненты просто принимают props и возвращают JSX, который описывает, как должен выглядеть UI. Эти компоненты **не имеют внутреннего состояния** и не могут изменять свое поведение в зависимости от событий или действий пользователя.

<CodeBlock code={`const Greeting = ({name}) => {
    return <h1>Hello, {name}!</h1>;
};`} language="javascript" />

Stateful компоненты **могут управлять состоянием** и изменять свое поведение в зависимости от событий. Они используют внутреннее состояние, чтобы отслеживать изменения и обновлять UI в ответ на действия пользователя или другие события.

<CodeBlock code={`import React, { useState } from 'react';

  const Counter = () => {
    const [count, setCount] = useState(0);

    const increment = () => {
      setCount(count => count + 1);
    };

    return (
      <div>
        <p>Count: {count}</p>
        <button onClick={increment}>Increment</button>
      </div>
    );
};`} language="javascript" />

Stateless компоненты просто рендерят UI, в то время как stateful компоненты имеют внутреннее состояние, которое позволяет им изменять своё поведение в ответ на события.

<Box type="warning" title="Важно!">
1. Держи компоненты маленькими и выполняющими одну задачу.
2. Названия компонентов всегда начинай с заглавной буквы.
3. Используй функциональные компоненты для всех новых проектов.
</Box>
